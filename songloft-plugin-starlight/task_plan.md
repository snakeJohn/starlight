# Starlight 2026-06-23 功能实施计划

## 目标

按照 `docs/superpowers/specs/2026-06-23-starlight-library-playback-registry-design.md` 实现列表/UI、Songloft 曲库接入、语音推送、换源、全局播放控件、插件源和版本自动化。

阶段 2：Songloft 数据接入，状态：进行中。

| 阶段 | 状态 | 交付内容 |
| --- | --- | --- |

## 验收清单

- 搜索、歌单、排行列表每页 20 条。
- 搜索结果有清空按钮、复选框和批量工具条。
- 单条歌曲 `播放` 按钮保留，并在本地全局播放控件完成后接入本插件播放队列。
- 长列表使用内部滚动容器。
- Songloft 歌曲库、歌单、本地歌曲可读取并推送音箱。
- 语音口令能匹配 Songloft 歌单和本地歌曲。
- 播放失败只在播放音源内换源。
- 下载失败只在下载音源内换源。
- 插件源符合仓库内 Raw URL 方式。
- 版本格式为 `V-yyyy.mm.dd.hh.mm`。

## 风险

- `static/js/music.js` 已经较大，阶段 1 尽量只做局部 helper 和渲染改动，后续全局播放控件阶段再考虑拆分。
- Songloft SDK 与宿主 HTTP API 返回结构可能不完全一致，阶段 2 需要兼容数组和分页结构。
- 下载源质量差异大，阶段 4 的错误信息必须清楚展示尝试过的源和最后失败原因。
